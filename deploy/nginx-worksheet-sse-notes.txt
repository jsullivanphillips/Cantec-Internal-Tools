Worksheet SSE endpoint (technician / staff session cookie auth):

  GET /api/monthly_routes/routes/<route_id>/worksheet/stream?month=YYYY-MM-01

Reverse-proxy checklist:

1. Disable response buffering for this path so events flush to the browser:
     proxy_buffering off;

2. Long-lived connection timeouts (adjust to taste):
     proxy_read_timeout 86400s;
     proxy_send_timeout 86400s;

3. The Flask handler sets header X-Accel-Buffering: no for nginx.

4. Do not set Connection on the Flask response (hop-by-hop; WSGI servers such as
   Waitress reject it). Configure keep-alive at the reverse proxy instead
   (e.g. proxy_http_version 1.1; proxy_set_header Connection '').

Example location snippet:

  location ~ ^/api/monthly_routes/routes/[0-9]+/worksheet/stream {
      proxy_pass http://your_upstream;
      proxy_http_version 1.1;
      proxy_set_header Connection '';
      proxy_buffering off;
      proxy_read_timeout 86400s;
      proxy_send_timeout 86400s;
  }

Each open worksheet tab holds one SSE connection; size worker concurrency accordingly.

Heroku / Waitress (Procfile):

- Default Waitress uses 4 threads; worksheet SSE tabs, ServiceTrade webhooks, and heavy
  ``refresh_paperwork`` GETs can exhaust the queue (``Task queue depth`` warnings, H12 timeouts).
- Procfile uses ``--threads=16`` so SPA HTML and API requests are less likely to wait behind
  long-lived SSE streams and deferred webhook work.
- Webhook handlers return immediately and run ServiceTrade sync on a background thread.
